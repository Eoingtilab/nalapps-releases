## 핵심 변경

### 이중 배포

- 한 번의 빌드에서 커머스용 애드온 5개와 독립 플러그인 5개 생성
- 기능 코드는 복사하지 않고 배포 진입점·라이선스·업데이트·메뉴 통합만 분리
- 두 배포형 동시 설치 시 제품별 sentinel과 고유 상수로 중복 실행 차단
- 여러 공식 애드온을 동시에 활성화해도 제품별 라이선스·업데이트 상태가 격리됨

### Core

- 애드온 실행코드 미포함
- 최초 설치 시 최소 하나의 애드온 선택
- 여러 개 또는 전체 선택
- WooCommerce 필요 제품 선택 시 설치 필요 안내와 명시적 동의 설치
- 대시보드·설정·라이선스·애드온·스토어 제공

### 라이선스

- 개발판은 시리얼 없이 기능 제한 없이 시험 가능
- EDD 운영판은 Core 및 제품별 라이선스 또는 Commerce Pro 필요
- 1년·평생 라이선스 구조
- 스킴·호스트·포트 바인딩으로 서브도메인 별도 인식
- 만료검사와 제한된 통신장애 유예

### 업데이트

- WordPress 표준 플러그인 업데이트 알림
- app.nal.la의 라이선스·상품정보와 GitHub Release의 ZIP·버전·SHA-256 역할 분리
- 다운로드 전 SHA-256 검증

## 알려진 제한사항

- 실제 app.nal.la EDD 상품 ID 미입력
- GitHub Release 자산이 아직 게시되지 않아 원격 자동 업데이트 미검증
- 실제 SMS·알림톡·SMTP 계정 미연결
- 실제 다양한 테마·캐시·보안 플러그인 조합 미검증
- RC이므로 운영 사이트 직접 설치 금지

- 다중 활성화와 충돌 차단 확인
- WooCommerce Classic·Blocks·HPOS 주요 흐름 확인
- VendorOps 혼합주문·출고·문의·반품·정산 확인
- 관리자 UI와 모바일 화면 확인

- 릴리즈 카탈로그·EDD·GitHub 업데이트 실연동 확인

## 롤백

- RC 설치 전 전체 파일·DB 백업
- 문제 발생 시 플러그인 비활성화 후 이전 버전 ZIP으로 파일 교체
- 애드온 데이터는 자동 삭제하지 않음
- DB 문제가 있으면 설치 전 백업으로 복원
